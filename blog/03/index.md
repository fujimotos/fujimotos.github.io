# [unix] Unixシェルで手軽にデータを集計する (前編)

クリアコードの藤本です。

私たちの業務では、よく「データを集計する」という仕事が発生します。
例えば、最近の改修の効果を測定するために、過去ログからリクエストの失敗レートを測定したり、
お客様への業務報告のために、チケットシステムから直近の稼働状況を集計したりします。

もちろん、同じようなタスクが定型的に発生するようであれば、専用のスクリプトを作り込むのですが、
この種類のタスクは単発限りで一度実行すればよい、ということが少なくありません。

筆者は、このようなタスクがある時は、Unixの標準ツール（とPython）を使って集計しています。
今回の記事では、具体的にどのように集計を行っているかを紹介したいと思います。

## 例: シェルでコマンドの実行回数を集計する

実際のタスクに適度に近い例として、「コマンドの実行履歴を集計する」というタスクを例に
集計のやり方を説明したいと思います。

bashであれば、集計対象のデータ（＝コマンドの実行履歴）は`history`で簡単に取得できます。

```bash
$ history
 ...
 1171  cd /var/log/td-agent
 1172  ls
 1173  less td-agent.log
 1174  less td-agent.log.1
 1175  grep -F '[warn]' td-agent.log.1
 ...
```

ここから「自分が最もよく実行しているコマンド」を集計してみましょう。
大雑把には、左から二番目の列を取り出して、コマンドごとに件数を数えればよさそうです
（注: ここではパイプやインラインのコマンドは無視しています）。

この処理の実現の仕方にはいくつかあるのですが、筆者であれば次のように打ち込んで集計します：

```bash
$ history | awk '{print $2}' | sort | uniq -c | sort -rn | head
    345 vim
    159 bundle
    101 ls
     52 cd
     48 grep
     39 ll
     32 cat
     31 history
     26 git
     23 rm
```

実行したコマンドの中身については後ほど解説するので、まずは出力の方をご覧ください。
左側がプログラムの出現回数で、右側が対応するプログラムの名前です（出現回数が多い順に並んでいます）。
vimがぶっちぎりで多く実行されている

皆さんも、試しに上のコマンドを自分の環境で実行してみると、自分が普段どのコマンドを使ってるのかが分かって楽しいと思います。

## これはどう動いているのか？

ご存知の方も多いとは思いますが、コマンドを`|`でつないでいるのは「パイプ」という仕組みです。
この`|`記号で連結することで、前のコマンドの処理結果を、後のコマンドに渡すことができます。
例えば、次のようにすると、`echo`の出力を`wc`（テキストの文字数その他を数えるプログラム）に渡せます：

```bash
$ echo -n aiueo | wc -c  # wc には"aiueo"という文字が渡される
5
```

パイプはいくつでも連結でき、実際の利用ケースでも何段も連結することが少なくありません。
実際、前節の例では、`history -> awk -> sort -> uniq -> sort -> head` という順番でデータを渡すのに利用しました。

普段何気なく使うことが多いのですが、パイプはUnixの偉大な発明の一つです。

### 構成要素(1) awkによるテキスト処理

プログラム全体の構成が分かったところで、今度はパイプラインの最初に登場する`awk`に注目しましょう。

`awk`はテキストデータを操作するための万能ツールです。本記事の関心の範疇で手短に説明するなら、
このプログラムは大雑把に次のような仕組みで動作します。

 1. 入力データから1行読み込む。
 2. 行データを空白文字で切っていき、内部的に配列として保持する
    (配列のそれぞれの要素は`$1`,`$2`という形で参照できる）
 3. 引数の命令処理を現在の行に適用する。
 4. 入力データから次の行を読み込む。
 5. ...

今回のコマンド集計では、単純に「各行から2番目の要素を取り出す」という用途に使っています。
自分で引数をいじってみると、感覚がつかめるのではないかと思います：

```bash
$ history | awk '{print $2}'
...
cd
ls
less
less
grep
...
$ history | awk '{print $1}'
...
1171
1172
1173
1174
...
$ history | awk '{print $1,$2}'
1171  cd
1172  ls
1173  less
1174  less
1175  grep
```

実は`awk`の実体はループや関数を備えた完全なプログラミング言語で、あらゆるタスクをこなすことができます。
実際、本記事の例では、いろいろなコマンドを組み合わせて処理を実現していますが、
awk単体で一連の処理を完結させることも可能です。

例えば、次のコマンドを実行すると、コマンドを集計できます：

```
$ history | awk '{A[$2]+=1} END{for (i in A) print i, A[i]}'
19 rm
21 w3m
13 sudo
21 git
50 cd
...
```

ここで「なぜ`awk`でできるのに、わざわざ他のコマンドを組み合わせるのか？」という疑問が思い浮かぶところですが、
筆者の考えでは次の二点が大きいです：

1. 本式のプログラムを書くのが面倒だからコマンドラインを使っているのに、awkで複雑なプログラムを書き始めるのは本末転倒。
2. データ集計は段階的に試行錯誤しながら行うことが多い。
   徐々にデータを変換していき、最終的に得たいデータを作る、というプロセスを伴うので、
   コマンドを適宜付け外しできるパイプラインの方が扱いやすい。

残りのコマンドについては後編で解説します。
